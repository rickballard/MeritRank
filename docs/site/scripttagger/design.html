ScripTagger: design notes

Summary-first, transcript optional

  • Default output is a two-panel summary: Best claims (well-sourced) and Questions to verify (needs evidence).
  • Transcript exists but is off by default; users can reveal/export it later.

Speaker inference from public posts

Most events are posted online. The app tries to infer the speaker via shared URL + page metadata (channel/author/title/date), and may add a speaker_hypothesis with a confidence score. Submissions remain observations until crowd consensus confirms.

Unverified → Consensus → Positive-only publication

score(dim, segment) = weighted_mean_i ( user_weight_i × evidence_weight_i × rating_i )

user_weight = f(RepTag⁺)          # positive-only component, bounded
evidence_weight = g(citations, independence, cross-agreement)
publish if: support ≥ threshold && independence ≥ min && no-policy-violations
  → mint positive credit (RepTag)     # negatives never publish

Consent & safety

  • “Assistance, not verdicts.”
  • Local-first; opt-in to contribute signed JSON to the append-only log.
  • Minimize PII; redact minors/sensitive info.
